##   superset发送报表到飞书

Clone 代码到本地

```
git clone git@github.com:apache/superset.git
```

如下是一些提示词和codex的反馈

```
下面的事情先讨论，不要着急动手。  我想要一个发送报表到飞书的功能，你有什么方案
```

... ... 

我建议先确认 5 个问题

  1. 你要发的是截图、明细文件，还是指标摘要？
  2. 触发方式是“定时”还是“阈值告警”？
  3. 发送对象是群机器人 webhook，还是点对点用户/部门？
  4. 是否允许引入一个中转服务？
  5. 对失败重试和审计留痕有没有合规要求？

  你确认这 5 点后，我可以给你一版更具体的技术设计（接口、任务流、失败处理、上线步骤）。

```
1. 发送截图。 2. 定时和 阈值告警都要。 3. 群机器人。 4. 不希望。 5. 暂无
```
