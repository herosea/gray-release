# AI 灰测数据库设计

> 创建时间：2025-12-24
> 更新时间：2026-03-02

## 1. 表结构

### 1.1 invite_code

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK AUTO_INCREMENT | 主键 |
| channel | VARCHAR(50) | 渠道名 |
| code | VARCHAR(6) UNIQUE | 邀请码（全局唯一） |
| source_type | VARCHAR(20) | 生成方式：`MANUAL`/`RANDOM` |
| total_quota | INT | 总额度 |
| used_quota | INT | 已使用额度 |
| alert_threshold | INT | 告警阈值 |
| status | VARCHAR(20) | 状态：`ENABLED`/`DISABLED` |
| deleted | TINYINT | 逻辑删除标识：0/1 |
| created_by | VARCHAR(50) | 创建人 |
| updated_by | VARCHAR(50) | 更新人 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

约束:
- `code` 全局唯一，历史码不可复用
- 逻辑删除不释放唯一约束

### 1.2 invite_redeem_log

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | BIGINT PK AUTO_INCREMENT | 主键 |
| invite_code_id | BIGINT FK | 关联邀请码 |
| user_id | VARCHAR(64) | 用户ID |
| redeem_result | VARCHAR(20) | 核销结果 |
| reason | VARCHAR(100) | 失败原因 |
| created_at | TIMESTAMP | 创建时间 |

说明:
- 当前版本主要用于审计预留；核销主流程后续前台联调时接入。

## 2. 核心口径

- 剩余额度：`remaining_quota = total_quota - used_quota`
- 告警判定：`remaining_quota < alert_threshold`
- 已使用判定：`used_quota > 0`
