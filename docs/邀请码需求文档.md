# 需求背景

- 邀请用户参与AI功能灰度测试，目标用户：
  - 对AI好奇，接受良好，愿意尝试新功能 
  - 记账新用户，记账习惯还在形成期，愿意尝试新功能 
  - 灰测目标：第一期100人？待定
- 用户来源渠道：
  - 自媒体（小红书）等
  - 公司内部
  - App内

# 需求范围

- 本次为了满足上述来源的用户参与灰测帮忙提提意见改进产品目的，提供发送统一固定的邀请码用于本次活动，用户来主动核销参与活动的方式
- 后续正式上线时，再考虑一人一码来达到用户体验功能福利，邀请机会稀缺性的目的（本期先不考虑）

### 一、邀请码逻辑

| 模块       | 功能       | 描述                                                         |
| :--------- | :--------- | :----------------------------------------------------------- |
|            | 统一开关   | 统一一个开关配置当前是否是灰测状态是：走下面邀请码逻辑否：则直接进入AI财务助手H5页面，不判断邀请码流程 |
| 邀请码发放 | 邀请码发放 | 本次邀请码活动固定的多个码 1223，用于运营发到站外自媒体或App内 |
|            | 邀请码属性 | 邀请码内容：6位数字和大写字母组合的邀请码，eg：E43AAB（技术可以参考，有更好的方式可以改）邀请码额度：支持配置体验额度（体验人数上限），本期先100？体验额度支持修改 邀请码额度不足告警：支持配置告警数量，比如告警数量10（则少于10个时进行告警）：符合小于10的条件就告警（相当于告警10次）。告警如何通知P1 1223 |
| 邀请码核销 | 邀请码核销 | App内AI财务助手H5页面：方式一：站外的用户输入邀请码核销来核销方式二：App内的用户无邀请码点击一键获取直接核销核销流程：核销成功后，toast提示：“恭喜您获得体验机会”；邀请码的额度 -1核销失败，toast提示具体原因，不扣减额度体验额度超限：“邀测名额已满，感谢关注” 1226邀请码错误提示：“邀请码无效” 1226兜底服务端异常提示：“服务异常” 1226 |
